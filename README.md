# تطبيق إدارة الورديات (Shift Management App) - نسخة Vercel

هذه النسخة معدلة خصيصاً لتعمل بكفاءة على منصة **Vercel** باستخدام قواعد بيانات سحابية دائمة، مما يحل مشكلة فقدان البيانات أو تعذر تسجيل الدخول.

## التغييرات الرئيسية للإصلاح:
1.  **نقل قاعدة البيانات:** تم تحويل تخزين البيانات من ملفات JSON المحلية (التي لا تعمل على Vercel) إلى قاعدة بيانات **MySQL** سحابية.
2.  **التوافق مع Serverless:** تم تعديل الخادم الخلفي ليعمل كدوال **Vercel Serverless Functions**.
3.  **إعدادات Vercel:** إضافة ملف `vercel.json` لتهيئة المسارات وبناء المشروع تلقائياً.

## متطلبات النشر على Vercel:

### 1. إنشاء قاعدة بيانات MySQL سحابية (مجانية)
نوصي باستخدام **TiDB Cloud** أو **Aiven** للحصول على قاعدة بيانات MySQL مجانية:
*   سجل في [TiDB Cloud](https://tidbcloud.com/).
*   أنشئ Cluster جديد (Free Tier).
*   احصل على رابط الاتصال (Connection String) الذي يبدأ بـ `mysql://...`.

### 2. إعداد المتغيرات البيئية (Environment Variables)
في إعدادات مشروعك على Vercel، أضف المتغيرات التالية:
*   `DATABASE_URL`: رابط الاتصال بقاعدة البيانات الذي حصلت عليه.
*   `JWT_SECRET`: أي نص عشوائي طويل لتأمين الجلسات.
*   `NODE_ENV`: قيمته `production`.

### 3. تهيئة قاعدة البيانات (مرة واحدة فقط)
بعد الحصول على رابط قاعدة البيانات، يمكنك تشغيل السكربت التالي محلياً لرفع بيانات الموظفين الأوليين إلى القاعدة السحابية:
```bash
# أضف DATABASE_URL في ملف .env محلياً أولاً
pnpm install
node setup-db.mjs
```

## طريقة النشر:
1.  ارفع الكود المعدل إلى مستودعك على GitHub.
2.  اربط المستودع بـ Vercel.
3.  سيقوم Vercel ببناء المشروع تلقائياً بناءً على ملف `vercel.json`.
