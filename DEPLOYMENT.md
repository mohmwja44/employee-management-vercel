# دليل النشر الشامل لتطبيق إدارة الورديات

هذا الدليل يوضح كيفية نشر تطبيق إدارة الورديات، وهو تطبيق كامل المكدس (Full-Stack) يتكون من واجهة أمامية (React) وخادم خلفي (Express).

## 1. المتطلبات الأساسية للنشر

لضمان النشر الناجح، يجب مراعاة ما يلي:

1.  **مستودع GitHub:** يجب رفع الكود كاملاً إلى مستودع GitHub خاص بك.
2.  **قاعدة بيانات دائمة:** التطبيق يتطلب قاعدة بيانات دائمة (Persistent Database) مثل **PostgreSQL** أو **MySQL**، حيث أن قاعدة بيانات SQLite المحلية (`sqlite.db`) غير مناسبة للنشر السحابي.
    *   **الخطوة:** قم بإنشاء قاعدة بيانات خارجية واحصل على **URL الاتصال** الخاص بها.
3.  **منصة استضافة:** نوصي باستخدام منصات تدعم تطبيقات Node.js الكاملة مثل **Render** أو **Railway**.

## 2. النشر عبر Render (موصى به)

Render هي منصة ممتازة لاستضافة تطبيقات Full-Stack Node.js.

1.  **رفع الكود:** تأكد من أن الكود مرفوع على GitHub.
2.  **إنشاء خدمة ويب (Web Service):**
    *   سجل الدخول إلى [Render.com](https://render.com).
    *   اختر **New > Web Service**.
    *   اربط مستودع GitHub الخاص بك.
3.  **إعدادات البناء والتشغيل:**
    *   **Root Directory:** (اتركها فارغة إذا كان الكود في الجذر)
    *   **Runtime:** `Node`
    *   **Build Command:** `pnpm install && pnpm build`
    *   **Start Command:** `pnpm start`
4.  **المتغيرات البيئية (Environment Variables):**
    يجب إضافة المتغيرات التالية (يمكنك استخدام واجهة Render لإضافتها):
    | المتغير | القيمة المقترحة | الوصف |
    | :--- | :--- | :--- |
    | `NODE_ENV` | `production` | لتشغيل التطبيق في وضع الإنتاج. |
    | `JWT_SECRET` | `مفتاح_سري_قوي_جداً` | مفتاح سري لتوقيع رموز JWT. **يجب أن يكون قوياً.** |
    | `DATABASE_URL` | `URL_قاعدة_بياناتك_الخارجية` | رابط الاتصال بقاعدة بيانات PostgreSQL أو MySQL. |
    | `OWNER_OPEN_ID` | `admin-id` | معرف المستخدم الذي سيتم تعيينه كمسؤول (Admin). |

## 3. النشر عبر Vercel (للواجهة الأمامية فقط)

إذا كنت تخطط لاستضافة الخادم الخلفي (Express) في مكان آخر (مثل Render) والواجهة الأمامية (React) على Vercel، اتبع الخطوات التالية:

1.  **نشر الخادم الخلفي أولاً:** اتبع خطوات Render للحصول على رابط للخادم الخلفي (API URL).
2.  **نشر الواجهة الأمامية على Vercel:**
    *   سجل الدخول إلى [Vercel.com](https://vercel.com).
    *   اختر **New Project** واربط مستودع GitHub.
    *   عند إعداد المشروع، قم بتعيين **Root Directory** إلى `client`.
    *   أضف متغير بيئة واحد:
        | المتغير | القيمة | الوصف |
        | :--- | :--- | :--- |
        | `VITE_TRPC_URL` | `رابط_الخادم_الخلفي_الذي_حصلت_عليه_من_Render` | رابط نقطة نهاية tRPC للخادم الخلفي. |

## 4. تجهيز الملفات لـ GitHub

لرفع المشروع على GitHub، ستحتاج إلى الملفات التالية:

*   جميع ملفات المشروع.
*   ملف `.gitignore` (موجود بالفعل) لضمان عدم رفع الملفات غير الضرورية مثل `node_modules` أو ملف قاعدة البيانات المحلية (`sqlite.db`).
*   ملف `.env.example` (تم إنشاؤه) لتوضيح المتغيرات البيئية المطلوبة دون كشف الأسرار.
*   ملف `README.md` (تم إنشاؤه) لتقديم المشروع.
*   ملف `DEPLOYMENT.md` (هذا الملف) لإرشادات النشر.

**الخطوة التالية:** قم بإنشاء مستودع جديد على GitHub ورفع جميع هذه الملفات إليه.
